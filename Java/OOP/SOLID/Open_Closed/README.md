##Принцип открытости/закрытости 

_Программные сущности должны быть открыты для расширения и закрыты для изменения._

Т.е. должна иметься возможность расширять поведение программных сущностей без их изменения

Рассмотрим пример применения принципа. Допустим мы можем выдать скидку клиенту 
```java
class Discount {
    giveDiscount() {
        return this.price * 0.2;
    }
}
```

Далее добавляется VIP клиент, для которого скидка больше. Внесем следующую модификацию:

```java
class Discount {
    giveDiscount() {
        if(this.customer == CustomerType.Normal) {
            return this.price * 0.2;
        }
        if(this.customer == CustomerType.Vip) {
            return this.price * 0.4;
        }
    }
}
```
Это является нарушением данного принципа, т.к. при появлении нового типа покупателя нужно будет вносить изменения в класс.

Пример кода с соблюдением принципа:
```java
class VIPDiscount: Discount {
    getDiscount() {
        return super.getDiscount() * 2;
    }
}
```
Таким образом при добавлении новых типов покупателей будет создаваться новый класс, расширяющий исходный
